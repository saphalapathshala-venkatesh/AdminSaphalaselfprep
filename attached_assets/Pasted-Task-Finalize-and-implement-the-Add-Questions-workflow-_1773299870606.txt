HIGH-LEVEL PRODUCT DECISION
“Imports” should no longer be treated as a separate standalone admin menu.
Instead, it should become an integrated feature inside Test Creation / Test Editing called:

ADD QUESTIONS

It should function as a section-wise / subsection-wise question ingestion and preparation workflow.

FINAL FEATURE GOAL
Inside Test Builder, admin should be able to:
1. Create/edit a test
2. Define sections
3. Optionally define subsections inside a section
4. For each section or subsection, click “Add Questions”
5. Add questions through multiple sources
6. Review, validate, edit, bulk edit, classify, and finalize questions
7. Save clean questions into Question Bank as new records where applicable
8. Attach selected clean questions to the current test section/subsection

IMPORTANT: The feature should feel premium, operationally practical, and safe for real admin usage.

PART 1 — TEST BUILDER STRUCTURE: SECTION + SUBSECTION SUPPORT
==================================================

Implement support for:
Test
→ Section
→ Subsection (optional)
→ Questions

Rules:
- A section may have zero or more subsections
- Questions may be added at section level or subsection level depending on builder design, but subsection-level targeting must be supported
- Subsections should be useful for structures like:
  - Reasoning → Seating Arrangement / Puzzles / Misc
  - Quant → Arithmetic / Algebra / DI
  - English → Comprehension / Cloze / Grammar

TIMER RULES FOR SUBSECTIONS
- Section has one total section time
- Subsection separate timer is OPTIONAL
- Some subsections may have separate timers while others may not
- Timed subsection totals must never exceed the parent section total time
- Untimed subsections should use the remaining shared section time pool
- Example:
  - Section total = 40 min
  - Subsection A timed = 10
  - Subsection B timed = 8
  - Remaining untimed subsection pool = 22
- The system should display:
  - Section total time
  - Timed subsection total
  - Remaining shared time
- Do NOT force all subsections to be timed
- Do NOT force equal split across untimed subsections
- Separate timer for subsection is optional and adaptive

Validation:
- If timed subsection totals exceed section total → blocking error, cannot save
- If mixed timed + untimed subsections exist → calculate and show remaining shared pool clearly

PART 5 — QUESTION CONTENT TYPES TO SUPPORT
==================================================

The system must support at least these import/review content structures:

A. Standalone single questions

B. Paragraph / passage / set-based grouped questions

Grouped question use cases include:
- English comprehension
- Reasoning seating arrangement
- Puzzle sets
- Data interpretation sets
- Shared-instruction question groups
- Caselet-type grouped questions

RULES FOR GROUPED QUESTIONS
- Shared paragraph / set / stimulus should be typed only once in the source template
- System should automatically associate that shared parent content with all child questions in the group
- There must be an explicit boundary / end indicator in upload format so the parser knows where the group ends
- This is necessary so the next single question or next new group is not incorrectly merged
- Group sizes can vary (2, 3, 5, 10, etc.)
- Order locking inside the group is NOT required; shuffling within group can be allowed later or in exam behavior
- But group relationships must remain intact
- In review UI, grouped questions should be clearly indicated as belonging to a shared parent/group

PART 9 — VALIDATION FIRST, NO DIRTY FINAL UPLOAD
==================================================

Upload/select flow must NOT blindly commit data.

The system should:
1. Parse input
2. Validate structure/content
3. Explain errors clearly
4. Allow admin fixes
5. Only after everything is clean enough, allow final upload/commit

Important:
All errors while uploading must be clearly explained.

Differentiate:
A. Blocking errors
- missing question text
- invalid options structure
- missing correct answer
- broken grouped structure
- invalid template format
- impossible timer/scoring structure

B. Warnings
- possible duplicate
- missing optional metadata
- questionable taxonomy
- missing explanation
- unused extra fields

Only after blocking issues are resolved should final upload be allowed.
