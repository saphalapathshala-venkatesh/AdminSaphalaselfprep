SUPPORTED IMPORT FORMATS:
A) CSV:
   Required columns for MCQ types:
   - type (MCQ_SINGLE|MCQ_MULTIPLE|DRAG_REORDER|DRAG_DROP|FILL_BLANKS|TRUE_FALSE)
   - stem
   - option1..option8 (optional depending on count)
   - correct (e.g., "1" for single, "1,3" for multi)
   - explanation (optional)
   - difficulty (FOUNDATIONAL|PROFICIENT|MASTERY)
   - status (DRAFT|APPROVED)
   - tags (comma separated)
   - category, subject, topic, subtopic (names; must map/create taxonomy)
   Validation rules:
   - stem required
   - type required
   - difficulty required
   - status required
   - For MCQ_*: at least 2 options, max 8; correct must be valid indexes
   - For non-MCQ: options/correct optional
   - Taxonomy mapping:
     * Category/Subject/Topic/Subtopic must resolve by name (case-insensitive)
     * If missing, create them (but only if ADMIN/SUPER_ADMIN; allowed)
     * Maintain hierarchy integrity

B) DOCX:
   - Minimal reliable parser: accept DOCX where each question is separated by blank line
   - Pattern:
     Q: ...
     A) ...
     B) ...
     C) ...
     D) ...
     Correct: B   (or "Correct: B,D")
     Explanation: ...
     Taxonomy: Category > Subject > Topic > Subtopic
     Difficulty: ...
     Status: ...
     Tags: ...
   - If parsing fails for a block, store it as invalid ImportRow with errorMsg.

API ENDPOINTS:
- POST /api/imports/preview
   Input: file (csv/docx)
   Output: ImportJob + first N rows preview (rawData), with initial validation results

- POST /api/imports/validate
   Input: importJobId
   Action: validate all rows using rawData/editedData; update ImportRow isValid/errorField/errorMsg; update ImportJob counts/status

- PUT /api/imports/rows/:id
   Input: editedData patch
   Action: save editedData for that row (do not import yet)

- POST /api/imports/revalidate
   Input: importJobId
   Action: re-run validation using editedData when present

- POST /api/imports/commit
   Input: importJobId
   Action:
     * import all valid rows into Question/QuestionOption (+ taxonomy create if needed)
     * update ImportJob status PARTIAL_IMPORTED or IMPORTED
     * generate error report CSV for invalid rows (store locally in /public/reports or similar) and set reportUrl

- GET /api/imports/:id
   Returns job + rows (paginated) with errors for UI
