2. Web device limit rule

Default policy:

one user may use only one active web device at a time

Interpretation:

one trusted browser/device slot by default

can be desktop browser or mobile browser

not both by default

Admin override:

allow selected users to access both laptop and mobile web

Recommended implementation:

store a numeric field such as maxWebDevices

default = 1

admin may set to 2 for approved users

Do not build Android device logic now.

3. Device enforcement behavior

When user logs in on web:

identify or register a trusted device/browser entry

if user is within allowed device count, allow login

if user exceeds allowed device count:

block login/session creation

return a clear message indicating device limit reached

suggest contacting admin if needed

This enforcement should work with the current auth/session flow.

Do not rely on impossible hardware-perfect identification; use a realistic trusted-device approach based on browser/device fingerprint inputs available server-side.
