API ROUTES TO IMPLEMENT:

1) /api/products (ProductPackage)
- GET /api/products?search=&active=&page=&pageSize=
- POST /api/products
- PUT /api/products/[id]
- DELETE /api/products/[id]
- POST /api/products/[id]/simulate-purchase
   Body: { userId, couponCode? }
   Action:
    - Load package
    - Optionally apply coupon if valid:
       * check date validity
       * check usage limits (global and per user)
       * check applicableEntitlements intersects package.entitlementCodes (if empty list => allow all)
    - Compute grossPaise, feePaise = round(gross*0.03), netPaise = gross-fee
    - Create Purchase row with stream based on package contents:
       if includes TESTHUB => stream TESTHUB else if includes SELFPREP_HTML => SELFPREP_HTML else etc (pick first matching in priority order)
    - Create/Upsert UserEntitlement rows for each entitlementCodes:
       ACTIVE, validUntil null, purchaseId = Purchase.id, tenantId null
    - Return purchase + entitlements created
   This is purely for admin testing and dashboard data; do not build payment gateway.
