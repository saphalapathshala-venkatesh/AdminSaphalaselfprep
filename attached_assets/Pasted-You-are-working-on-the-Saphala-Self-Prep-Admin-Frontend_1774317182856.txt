Current architecture context

The current taxonomy chain is:

Category
Subject
Topic
Subtopic

Right now, Subject is category-scoped, which forces duplicate subjects like “History” under APPSC and AP Police. The architecture note recommends the safe additive fix: add a CategorySubject junction table instead of refactoring existing Subject ownership.

Also note: much of the content already stores taxonomy loosely by IDs/strings, so this must be implemented as a safe retrieval/filtering improvement, not as a destructive remap.

Required implementation
Part 1 — DB schema (additive only)

Add a new Prisma model:

model CategorySubject {
  id         String   @id @default(cuid())
  categoryId String
  subjectId  String
  createdAt  DateTime @default(now())

  category   Category @relation(fields: [categoryId], references: [id], onDelete: Cascade)
  subject    Subject  @relation(fields: [subjectId], references: [id], onDelete: Cascade)

  @@unique([categoryId, subjectId])
  @@index([categoryId])
  @@index([subjectId])
}
Important
