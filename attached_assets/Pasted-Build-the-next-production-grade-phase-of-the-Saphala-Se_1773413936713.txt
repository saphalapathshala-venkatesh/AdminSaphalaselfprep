Build the next production-grade phase of the Saphala Self Prep / Admin system for student frontend, structured courses, reusable content items, premium learning UI, E-Books, Flashcards, progress tracking, protected content access, and detectable infringement monitoring, using the existing stack and current live schema as safely as possible.

Follow the already locked architecture direction:

do not make unnecessary schema changes

do not make disruptive refactors unless truly required

keep DB changes minimal

extend current architecture carefully and cleanly

build reusable components and services where possible

The result must feel like a premium EdTech product, not a generic admin CMS or file manager.

1) Core course architecture

Implement a structured course system organized as:

Category → Subject → Chapter → Lesson → Content Items

Supported content types now:

Video

E-Book

PDF

Flashcard Deck

Architecture should remain extensible for later item types.

Important structural rules

Every course must first be assigned to a Category

Inside a course, Subject is the first visible learning grouping

Subject and Chapter are structural grouping layers

Lesson is the actual content layer

Actual study items live inside lessons

2) Product naming update

Across admin UI, student UI, course builder, lesson item lists, content cards, and icons:

rename HTML Material to E-Book

21) Quiz Card behavior

22) Typed Fill-in-the-Blank Card

This must be a separate card type.

Behavior:

learner types answer manually

clicks Submit Answer

system validates

if correct, Flip Card becomes available

if wrong, allow retry

after flip, show explanation

Critical requirement

For each blank, admin must be able to define:

one canonical answer

multiple accepted answer variants (e.g. 4–5 valid forms)

Validation rules

trim spaces

lowercase by default

collapse extra spaces

optionally ignore punctuation where appropriate

deterministic comparison only against admin-approved accepted variants

Support:

single blank now

architecture should be extendable to multi-blank
