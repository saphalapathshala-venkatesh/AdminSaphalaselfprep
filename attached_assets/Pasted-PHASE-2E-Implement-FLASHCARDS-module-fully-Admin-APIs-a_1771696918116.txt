API ROUTES TO CREATE (App Router route handlers):
1) GET  /api/flashcards/decks
   - Query: page, pageSize, search (title), isPublished, categoryId/subjectId/topicId/subtopicId (filter decks by their cards’ subtopic taxonomy if provided)
   - Returns: {items, total, page, pageSize}

2) POST /api/flashcards/decks
   - Body: {title, description?, categoryId?, subjectId?, topicId?}
   - Creates deck with createdById from session
   - Audit: FLASHCARD_DECK_CREATE

3) PUT /api/flashcards/decks/[id]
   - Body: {title?, description?, categoryId?, subjectId?, topicId?, isPublished?}
   - Toggle publish via isPublished (set true/false)
   - Audit: FLASHCARD_DECK_UPDATE and additionally FLASHCARD_DECK_PUBLISH or FLASHCARD_DECK_UNPUBLISH when isPublished changes

4) DELETE /api/flashcards/decks/[id]
   - If deck.isPublished and role != SUPER_ADMIN => 403
   - Otherwise delete deck (cascade should delete cards via relation)
   - Audit: FLASHCARD_DECK_DELETE

5) GET  /api/flashcards/decks/[id]/cards
   - Query: page, pageSize, search (front/back), subtopicId
   - Order by "order" ascending
   - Returns list + total

6) POST /api/flashcards/decks/[id]/cards
   - Body: {front, back, imageUrl?, subtopicId?}
   - Auto assign order = (max order in deck + 1)
   - Audit: FLASHCARD_CARD_CREATE

7) PUT /api/flashcards/cards/[id]
   - Body: {front?, back?, imageUrl?, subtopicId?}
   - Audit: FLASHCARD_CARD_UPDATE

8) DELETE /api/flashcards/cards/[id]
   - Audit: FLASHCARD_CARD_DELETE

9) PUT /api/flashcards/decks/[id]/reorder
   - Body: {orderedCardIds: string[]}
   - Update all cards’ order fields in a Prisma transaction
   - Validate: orderedCardIds must match all existing card ids for that deck (same count)
   - Audit: FLASHCARD_CARD_REORDER
