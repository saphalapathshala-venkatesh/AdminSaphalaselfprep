FEATURE 1 — Course Validity Options

Add flexible validity configuration in the Course create/edit form.

Admin should be able to choose one of:

Course Validity Type

( ) Days
( ) Months
( ) Custom End Date

If Days selected:
Input field → validityDays

If Months selected:
Input field → validityMonths

If Custom End Date selected:
Calendar date picker → validUntil

IMPORTANT BUSINESS RULE:

Course validity must always be calculated from the STUDENT PURCHASE DATE,
not from course publish date.

Example:

Course validity = 90 days
Student purchases on: 10 March
Course expires for that student on: 8 June

Another student purchases same course on: 1 April
Their expiry becomes: 30 June

Meaning:
Validity must be applied individually per purchase.

Therefore:

validUntil = purchaseDate + validityDays OR validityMonths

If Custom End Date is used:
That date is absolute and applies to all users.

Database expectations:

Add minimal fields needed to store validity configuration safely, for example:

validityType
validityDays
validityMonths
validUntil

Existing courses must remain valid and unaffected.

If validity is not configured:
Existing behavior should continue (treated as unlimited or current default).

Update Course create/update APIs to accept these fields.

FEATURE 2 — Test Scheduled Start DateTime

Add a field in Test creation/edit:

Label:
Test Start Time

Type:
Calendar + Time picker (datetime)

Purpose:

Even if the test is Published,
students cannot attempt the test until the selected start time.

Database field example:

testStartTime (datetime)

Example:

Test published today
Start time set to:
10 June 2026
08:00 PM

Students will see the test but cannot attempt until that time.

Student logic later will be:

NOW >= testStartTime

Until then:
Attempt button should remain disabled.

IMPORTANT:

Test expiration should NOT be stored separately.

Test availability automatically ends when the student's course validity expires.

Existing tests must remain unaffected if no start time is configured.
