------------------------------------------------

BACKGROUND

We are planning to introduce a Video Module in Saphala Self Prep.

Videos will be used for structured course lessons and must integrate with the existing taxonomy system:

Category
→ Course
→ Subject
→ Topic
→ Subtopic

Videos must NEVER be downloadable.

Students must only be able to stream videos.

------------------------------------------------

PROPOSED VIDEO STACK

Encoding + Storage + CDN

Bunny Stream

Video Player

Video.js

Streaming Protocol

HLS Adaptive Streaming

Live Classes

Zoom

------------------------------------------------

LIVE CLASS WORKFLOW

Live class hosted on Zoom.

After the class:

Zoom recording
→ downloaded
→ uploaded to Bunny Stream
→ encoded into VOD
→ attached to course

------------------------------------------------

RECORDED VIDEO WORKFLOW

Admin creates video lesson.

Steps:

1. Select taxonomy
   category
   course
   subject
   topic
   subtopic

2. Upload video

3. Video sent to Bunny Stream

4. Bunny encodes adaptive formats

5. System stores provider_video_id

6. Admin preview

7. Publish video

VIDEO SECURITY REQUIREMENTS

Videos must not be downloadable.

Planned protections:

• HLS streaming only
• token playback
• domain restriction
• no direct file URLs
• disable raw download links

Possible future improvements:

• watermarking
• playback analytics
• session-based access tokens

------------------------------------------------

PLAYER ARCHITECTURE

Frontend player:

Video.js

Requirements:

• mobile compatibility
• subtitle support
• analytics support later

ADMIN VIDEO MANAGEMENT
