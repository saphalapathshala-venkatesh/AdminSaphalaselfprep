Build the next production-grade phase of the Saphala Self Prep / Admin system for course structure, content mapping, premium learning UI, progress tracking, and protected content access, using the existing stack and current live schema as safely as possible. Follow the already locked architecture direction: do not make unnecessary schema changes or disruptive refactors. Keep DB changes minimal and only add what is truly required.

Core goal

Implement a structured course builder and student learning experience where a course is organized as:

Category → Subject → Chapter → Lesson → Content Items

This must work for these content types:

Video

HTML Material

PDF Material

Flashcard Deck

The result must feel modern, premium, clean, and unique, not like a generic file manager or dull admin tree.

1) Course creation structure

Update course creation/editing so that every course must first be assigned to a Category.

Once a category is selected, the course should support this internal learning structure:

Category

Subject sections / subject folders

Chapters

Lessons

Content items

Important:

Subject is the first visible grouping under a course

Subject UI should look like a premium learning module card/folder, not a standard computer folder icon

Chapters and lessons should be clearly nested and easy to navigate

Lessons are the actual content layer

Subject and chapter are structural headers/grouping layers

Actual study items live inside lessons

2) Product-type-aware course structure

During course creation, the admin already selects product types such as:

HTML Course

Video Course

PDF Course

Flashcard support

Test Series, etc.

Use those selected product types to control what item types can be added inside lessons.

Example:
If a course is configured with:

Video

HTML

PDF

Flashcards

then inside lessons, admin should be able to add:

video items

HTML material items

PDF material items

flashcard decks

If a product type is not enabled for that course, do not show that item type in the lesson item picker unless course config is later updated.

3) Course overview UI — subject folders

At the course level, subjects should be displayed as clean module cards / subject folders, not traditional folder icons.

Each subject folder card should include:

subject icon

subject color accent

subject title

optional short subtitle/descriptor

completed item count (example: 10 / 250 items completed)

progress percentage

progress bar

click/open behavior

Visual direction

These subject cards must feel premium:

rounded corners

subtle shadow

subject-color accent strip or border

soft hover/active states

clean spacing

strong readability

no Windows/macOS-style folder visuals

Use a polished card-based learning UI.

4) Subject-inside-course layout

When the learner opens a subject, use a split layout:

Left sidebar = chapter and lesson navigation

Right content area = current lesson details and content items

This must work beautifully on the web and degrade responsively on smaller screens.

5) Left sidebar learning navigator

The left sidebar must feel like a learning navigator, not a raw nested file tree.

Inside a subject:

show subject title at top

show subject progress summary (e.g. 18 / 52 completed)

show chapters as clean accordion sections

show lessons as elegant rows under expanded chapters

active lesson should be visibly highlighted

completed lessons should show green tick/completed state

incomplete lessons should show neutral state

chapter progress should be visible in a subtle way

Example feel:

modern accordion navigation

lesson list with checkmarks/status indicators

active lesson pill/card highlight

no clunky nested border tree UI

6) Chapter structure

Inside each subject, admin must be able to create multiple chapters.

Each chapter should support:

chapter title

optional short description

manual ordering

expand/collapse

item completion summary

Example:

Chapter 1: Fundamental Rights

2 / 10 items completed

Chapters are organizational, not the final content unit.

7) Lesson structure

Inside each chapter, admin must be able to create multiple lessons.

Lessons are the actual content layer.

Each lesson should support:

lesson title

optional short description

manual ordering

published/draft status if needed

content item count

progress tracking based on underlying items

Inside each lesson, admin can add the actual content items.

8) Content items inside lesson

Inside a lesson, admin should be able to add content items depending on selected product types for the course.

Supported item types now:

Video

HTML Material

PDF Material

Flashcard Deck

Architecture should remain extensible.

Lesson item capabilities

Each lesson should allow:

add existing item from library

create new item where appropriate

reorder items manually

edit attached item

remove item from lesson without deleting the underlying library item

Every learning item must first belong to taxonomy, then may optionally be attached to one or more courses/lessons.

This should apply to:

Video

HTML Material

Flashcard Deck

Each library item should support:

Subtopic (optional/recommended)

title

content-specific fields

edit option

publish/unpublish where relevant

Then that item can be attached to lesson(s) inside course(s).

Removing from a lesson must not delete the underlying content item.

14) Progress tracking and completion

When a learner completes an item, that item must be marked completed visually and the system must roll progress upward automatically.

Completion rollup

Item completed
→ lesson progress updates
→ chapter progress updates
→ subject progress updates
→ course progress updates

Examples

At subject level:

10 / 250 items completed

At chapter level:

Chapter 1 — 2 / 10 items completed

At course level:

overall progress percentage

If more items are added later, counts and percentages must recalculate dynamically and correctly.

No stale hardcoded progress.

21) Protected Content Access Gate

Implement a reusable Protected Content Access Gate before opening any protected:

HTML material

Flashcard deck

protected PDF

This must not be just a frontend popup. It should be backend-recorded and meaningful.

Modal/dialog content

Use wording close to:

Protected Content Notice
This content belongs to Saphala Pathshala and is provided only for your personal learning use.
Copying, sharing, recording, downloading, redistributing, or publishing this content without permission is prohibited.
All your activities inside Saphala may be monitored for security and content protection purposes. If misuse or redistribution is detected, your account may be suspended or blocked as per platform policy.

Checkbox:

I have read and agree to the Terms & Conditions

Buttons:

Accept and Continue

Cancel

User must explicitly accept before content loads.

Important:

do not preload full HTML material before acceptance

do not fetch full flashcard deck before acceptance

do not open/generate protected PDF URL before acceptance
