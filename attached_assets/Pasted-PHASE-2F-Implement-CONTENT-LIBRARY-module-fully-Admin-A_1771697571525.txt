API ROUTES TO IMPLEMENT:

CONTENT PAGES (HTML)
1) GET /api/content-pages
   - Query: page,pageSize,search,title, isPublished, categoryId/subjectId/topicId/subtopicId (filter)
   - Return: {items,total,page,pageSize}
2) POST /api/content-pages
   - Body: {title, body, subtopicId?}
   - If subtopicId set, also store derived topicId/subjectId/categoryId fields in the record? (Note: schema for ContentPage only has subtopicId; so only store subtopicId. Filtering by parents can be done via joins.)
   - Set createdById from session
   - Audit: CONTENTPAGE_CREATE
3) PUT /api/content-pages/[id]
   - Body: {title?, body?, subtopicId?, isPublished?}
   - When isPublished flips true: set publishedAt=now
   - When isPublished flips false: publishedAt=null (or keep last publishedAt; choose deterministic: set null)
   - Audit: CONTENTPAGE_UPDATE and CONTENTPAGE_PUBLISH/CONTENTPAGE_UNPUBLISH when publish changes
4) DELETE /api/content-pages/[id]
   - If published and role != SUPER_ADMIN => 403
   - Audit: CONTENTPAGE_DELETE

PDF ASSETS
5) GET /api/pdf-assets
   - Query: page,pageSize,search,isPublished,taxonomy filters similar to above
6) POST /api/pdf-assets
   - Accept multipart/form-data with:
     - title
     - file (pdf)
     - optional taxonomy ids: categoryId, subjectId, topicId, subtopicId
   - Validate mimeType pdf, size limit (e.g., 20MB)
   - Save file under /public/uploads/pdfs
   - Save PdfAsset row with fileUrl, fileSize, mimeType, createdById
   - Audit: PDFASSET_CREATE
7) PUT /api/pdf-assets/[id]
   - Accept JSON for metadata updates: {title?, taxonomy ids?, isPublished?}
   - Optionally support replacing file via multipart endpoint /api/pdf-assets/[id]/replace (optional only if easy)
   - Publish sets publishedAt=now, unpublish clears publishedAt
   - Audit: PDFASSET_UPDATE and PDFASSET_PUBLISH/PDFASSET_UNPUBLISH
8) DELETE /api/pdf-assets/[id]
   - If published and role != SUPER_ADMIN => 403
   - Delete DB row; optionally delete file from disk if exists
   - Audit: PDFASSET_DELETE
