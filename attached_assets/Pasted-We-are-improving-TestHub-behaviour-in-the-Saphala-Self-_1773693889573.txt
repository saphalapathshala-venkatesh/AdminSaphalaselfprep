FEATURE SET

1. Strict Section Lock Mode

Add checkbox in test creation:

"Lock sections until time ends or section is submitted"

Behaviour when enabled:

- learner cannot jump to next section early
- learner cannot revisit previous section
- section must be submitted or time must expire
- after section submission learner automatically moves to next section
- previous section becomes inaccessible

--------------------------------------------------

2. Section Submit Feature

Implement "Submit Section".

Behaviour:

Submit Section button appears when strict mode enabled.

When clicked:
- close current section
- persist answers
- advance learner to next section
- previous section becomes locked

If last section submitted → proceed to full test submit.

--------------------------------------------------

3. Timer Validation (CRITICAL FIX)

Currently system allows:

sectionDuration > totalDuration

This must never happen.

Add validation in test creation:

Rules:

totalDuration ≥ sum(sectionDurations)

If violated:

Block test save with error:

"Total test duration must be greater than or equal to the sum of section durations."

Also validate:

sectionDuration must be positive.

--------------------------------------------------

4. Pause Test Backend Support

Frontend shows "Failed to pause".

Audit pause implementation.

Ensure routes exist:

POST /attempts/{id}/pause
POST /attempts/{id}/resume

Pause must:

- create AttemptPause record
- change AttemptStatus → PAUSED
- freeze timers

Resume must:

- close pause event
- extend endsAt by pause duration
- change AttemptStatus → IN_PROGRESS

--------------------------------------------------

sectionDuration

Frontend uses these to render behaviour.
