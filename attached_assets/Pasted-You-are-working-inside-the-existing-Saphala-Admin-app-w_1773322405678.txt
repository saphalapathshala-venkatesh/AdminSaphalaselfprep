You are working inside the existing Saphala Admin app, where Videos and LIVE Classes modules have already been implemented.

This task is not about Bunny, Zoom, or student frontend.
Do not integrate external providers in this prompt.
Do not redesign existing modules unnecessarily.
Do not perform broad schema rewrites.

Goal

Implement admin-controlled exact content flow ordering for learning content.

Admin must be able to arrange content in the exact sequence they want for a learner, across mixed content types:

Videos

PDFs / content library materials

Flashcard decks

This ordering is for the teaching/learning flow, not just sorting by upload date.

Product requirement to follow

A course/topic/subtopic should support a unified sequence like:

Video — Introduction

PDF — Notes

Flashcards — Key Terms

Video — Deep Explanation

PDF — Revision Sheet

Flashcards — Final Recall

Admin must be able to control this order manually.

This is a locked requirement.

Main architecture decision

Do not build separate ordering systems for:

videos

PDFs

flashcards

Instead, build one unified ordered flow that can contain mixed content types.

Expected admin behavior

Inside an admin page for a selected context, admin should be able to:

choose a parent context such as:

category

course

subject

topic

subtopic

view all linked content items across:

videos

PDFs/materials

flashcard decks

arrange them in exact order

save the order

later edit/rearrange it without re-uploading anything

Implementation goal

Create a new admin feature/page called something like:

/admin/content-flow
or

/admin/course-flow

Use the naming that best fits the current codebase.

This page should become the central place to manage ordered learning flow.

Required approach
1. Add unified flow entity / mapping

If schema support is needed, add a minimal safe model for ordered content flow entries.

Recommended concept:

one parent context

one row per ordered item

each row stores:

tenantId

categoryId / courseId / subjectId / topicId / subtopicId as applicable

contentType (VIDEO, PDF, FLASHCARD)

contentId

displayOrder / sortOrder

optional title snapshot for easier rendering if useful

timestamps

Do not overengineer with polymorphism that breaks Prisma ergonomics.
Keep it simple and production-safe.

If the existing schema already has a suitable content-mapping model, reuse or extend it instead of inventing duplicate systems.

2. Parent context behavior

Ordering should work within a selected context.

Preferred behavior:

admin selects:

Category

Course

Subject

Topic

Subtopic

system loads relevant flow items for that exact context

admin reorders within that context only

If the codebase already uses a course-first structure, adapt cleanly, but keep topic/subtopic filtering available wherever practical.

3. Mixed content list UI

Build a clean ordered list with mixed content items.

Each row should show:

drag handle if implemented

content type icon/badge

thumbnail if available

title

current order number

actions:

move up

move down

remove from flow

open/edit source content

4. Reordering UX

drag-and-drop reorder

move up / move down buttons

sequence number editing

save order button

Persist order safely.

Add Video

Add PDF / Material

modal selectors

side panel

6. Remove from flow

Admin should be able to remove an item from the ordered flow without deleting the underlying source content.

Important rule:

removing from flow must not delete the actual video, PDF, or flashcard deck

it should only remove the mapping/order entry

Store order explicitly with a numeric field like:

displayOrder
or

sortOrder

renumber safely

avoid duplicates
